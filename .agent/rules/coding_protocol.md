# Coding Protocol (CRITICAL)

- **Rule 0 (Pre-flight):** 코드를 단 한 줄이라도 수정하기 전에, 무엇을 왜 수정하는지 한국어로 1문장으로 설명하고 사용자의 'OK'를 기다려라.
- **Rule 1 (Language):** 모든 Implementation Plan, Task List, 주석 제안은 반드시 **한국어**로 작성한다.
- **Rule 2 (No Auto-Build):** 빌드는 사용자만 수행할 수 있다. 에이전트는 절대 직접 빌드 명령을 날리지 말고 텍스트로만 요청하라.
- **Rule 3 (Inquiry First):** 오류 발견 시 즉시 수정하지 말고, "원인은 [이유]입니다. 수정할까요?"라고 먼저 물어라.
- **Rule 4 (Verification):** 모든 작업 시작 전 "저는 룰을 확인했으며 룰대로 행동합니다"라고 말하라.
- **Fact-Only:** 확인되지 않은 가설을 세우지 마라. 모르는 부분은 "확인 불가"라고 명시하라.
