# 구현 계획서: 재사용 가능한 채팅 UI 및 통합 커맨드 시스템 연동

**작성일**: 2026-03-04
**상태**: 진행 중 (Planning)

## 1. 개요
서버에 구현된 '하이브리드 통합 커맨드 시스템'을 인게임에서 편리하게 사용할 수 있도록, 로비와 게임 씬 모두에서 재사용 가능한 채팅 UI를 구축하고 연동합니다.

## 2. 현재 상황 분석
- **서버**: `CommandManager`를 통해 터미널과 채팅 명령어가 통합되어 관리가 일원화됨. `/add_weapon`, `/spawn` 등의 명령어가 준비됨.
- **클라이언트**:
    - `S_Chat` 패킷에 대한 핸들러가 등록되어 있지 않음.
    - 채팅을 입력하거나 표시할 수 있는 UI 컴포넌트(`InputField`, `ScrollRect`)가 부재함.
    - 로비와 게임 씬에서 각각 채팅이 가능해야 하나 로직이 파편화될 우려가 있음.

## 3. 상세 구현 계획

### 3.1 채팅 UI 컴포넌트 개발 (`ChatUI.cs`) [NEW]
- **위치**: `Assets/_Project/Scripts/UI/ChatUI.cs`
- **주요 기능**:
    - `InputField` 엔터키 입력 시 `C_Chat` 패킷 전송.
    - 서버로부터 수신한 `S_Chat` 메시지를 하단 `ScrollRect` 내의 `Text` 리스트로 출력.
    - 명령어('/')로 시작하는 메시지는 별도의 강조 효과 제공 (선택 사항).
    - 씬 전환 시에도 파괴되지 않거나(DontDestroyOnLoad), 각 씬의 프리팹으로 존재하여 동일한 로직 공유.

### 3.2 패킷 핸들러 및 매니저 업데이트 [MODIFY]
- **PacketHandler.cs**: 
    - `Handle_S_Chat(IMessage packet)` 메서드 추가.
    - 수신된 메시지를 `ChatUI` 인스턴스에 전달하여 화면에 표시.
- **PacketManager.cs**:
    - `MsgId.SChat`에 대한 파서(`MakePacket<S_Chat>`)와 핸들러(`PacketHandler.Handle_S_Chat`) 등록.

### 3.3 씬 통합 (Unity Editor)
- **로비(Lobby)**: 하단 혹은 우측에 채팅창 프리팹 배치. 로비 상태에서는 전체 채팅(LobbyChat MQ) 연동 확인.
- **게임(Game)**: HUD의 일부로 채팅창 배치. 게임 중 콘솔 명령어를 통해 실시간 디버깅 가능하도록 구성.
